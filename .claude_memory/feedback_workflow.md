User bevorzugt Multi-Agent-Workflow mit Rollen:
1. Business Analyst / Product Owner (Konzeption)
2. Software Engineer / Architect (technisches Design)
3. Dev/Ops Team (Implementierung)

**Why:** User will Context-Window schonen durch Subagents und klare Rollentrennung.
**How to apply:** Bei komplexen Tasks immer Subagents fuer verschiedene Perspektiven einsetzen.

Weitere Praeferenzen:
- Keine Multiple-Choice-Fragen, lieber offenes Gespraech
- Ausfuehrliche Konzeptdokumente als HTML (nicht nur Markdown)
- Alle Dateien in GitHub sichern bevor zur naechsten Phase uebergehen
- Gruendlich alle Quelldateien lesen bevor Konzept erstellen
